# 外贸开发客户规则库

这套规则把外贸开发客户经验沉淀为系统可执行的流程。新用户只需要输入关键词和目标平台，系统按优先级把搜索结果沉淀为线索资产，再决定是否进入开发动作。

## 优先实现顺序

1. 信息资产补全：先补官网、邮箱、国家，再补 LinkedIn、员工规模、创始人、融资情况。
2. 客户质量分层：按行业匹配、年采购量、供应商痛点、决策人可达性、项目紧急度评分。
3. 淘汰低价值客户：零售、纯贸易倒手、只比价不下单、无项目场景的客户进入 C 级，不消耗人工时间。
4. 新用户执行引导：S/A 级先开发，B 级观察，C 级放弃或仅自动化低频触达。
5. 跟进 SOP：第 0、3、7、14、30 天按不同触点推进，始终围绕项目、交期、现货、规格适配，而不是先报价。

## 资产字段规则

| 字段 | 用途 | 系统动作 |
| --- | --- | --- |
| 公司名称 | 统一客户资产主键 | 清理账号名、平台噪音、LinkedIn 掩码文本 |
| 官网 | 判断是否真实企业、提取邮箱和国家 | 优先从搜索结果和 profile_url 中识别非社媒域名 |
| 创始人 | 判断决策人线索 | 从官网 About/Team 文案和公开资料抽取 |
| LinkedIn | 复核公司或决策人身份 | 缺失时搜索 LinkedIn company/in 页面 |
| 邮箱 | 开发动作入口 | 从官网、联系页、搜索结果中抽取公开业务邮箱 |
| 融资情况 | 判断阶段和采购可能性 | 抽取 seed/series/funded/bootstrapped 等信号 |
| 员工规模 | 判断采购能力 | 抽取 employees/headcount/company size |
| 国家 | 判断目标市场匹配 | 从域名、地址、页面文本推断 |

## 开发客户判断

优先开发：
- 工程分销商、项目分销商、电气/照明分销商。
- 亮化工程、户外照明、装修、fit-out、项目承包商。
- 有项目采购、年度采购、交期、库存、供应商问题信号的客户。

不优先开发：
- 零售店、消费者小店、dropshipping。
- 纯贸易中间商、broker、middleman。
- 只询价、只比价、没有项目和采购节奏的客户。

## 出海开发话术原则

第一句话必须提到客户的具体项目、产品或业务信号。不要用空泛寒暄，不先报价格。结尾引导客户确认交期、现货或规格适配，让对话进入真实采购判断。

机器可读规则位于 `config/customer_development_rules.json`，后端会把规则版本、字段完整度、评分、下一步动作写入每条线索的 `metadata`。
