# 代码审查子代理

## 使用场景
对大型代码变更（超过 5 个文件或 500 行以上）进行审查时使用此子代理。

## 职责

1. **代码质量检查**
   - 命名规范和代码风格
   - 函数复杂度
   - 重复代码
   - 缺少的类型提示

2. **安全漏洞检测**
   - SQL 注入风险
   - XSS 漏洞
   - 敏感信息泄露
   - JWT 安全配置

3. **性能问题**
   - N+1 查询
   - 同步阻塞操作
   - 内存泄漏

4. **最佳实践**
   - FastAPI 路由设计
   - 数据库连接管理
   - 错误处理

## 审查重点

### Python Backend (backend/)
- `api/` - 路由是否有正确的错误处理和类型提示
- `db.py` - 是否使用连接池、UPSERT、事务
- `middleware.py` - 中间件是否正确配置

### Agent 代码 (agents/)
- `base_agent.py` - 抽象基类是否被正确继承
- 各 Agent - run() 方法是否使用 async/await

### 前端 (frontend/src/)
- React 组件是否有正确的状态管理
- API 调用是否有错误处理

## 输出格式

```

1. 添加类型提示以提高代码可维护性
2. 考虑使用 Pydantic 模型验证请求参数
```
